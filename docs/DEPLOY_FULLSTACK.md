# Full-stack deployment (API + UI)

Architecture:

| Layer | Role | Suggested host |
|-------|------|----------------|
| **Frontend** | Static Vite build (React) | [Vercel](https://vercel.com) |
| **Backend** | FastAPI + Uvicorn | [Render](https://render.com), [Railway](https://railway.app), [Fly.io](https://fly.io), or any Docker host |

The app is **not** a single Vercel serverless project: the Python API must run as a long-lived web process with enough RAM for `pandas` / `scikit-learn`.

---

## Prerequisites

- GitHub repo connected to your hosts (e.g. [Vikram-git/house-price-prediction](https://github.com/Vikram-git/house-price-prediction)).
- After both deploys, you will have:
  - **API URL** â€” e.g. `https://house-price-api.onrender.com`
  - **UI URL** â€” e.g. `https://house-price-platform.vercel.app`

## Step 1 â€” Deploy the API (Render, recommended)

### Option A â€” Blueprint (`render.yaml`)

1. In [Render Dashboard](https://dashboard.render.com) â†’ **New** â†’ **Blueprint**.
2. Connect the repo and select the branch.
3. Render reads `render.yaml` at the repo root.
4. After the service is created, open **Environment** and set:
   - **`CORS_ORIGINS`** â€” your final Vercel URL(s), comma-separated, e.g.  
     `https://your-app.vercel.app`  
     (You can add this after Step 2, then redeploy.)
   - **`DEMO_PASSWORD`** â€” choose a strong password (optional but recommended).

`SECRET_KEY` is generated by the blueprint (`generateValue: true`).

5. Copy the service **URL** (e.g. `https://house-price-api.onrender.com`).

### Option B â€” Docker (Render / Railway / Fly)

From the **repository root**:

```bash
docker build -t house-price-api .
docker run -p 8000:8000 \
  -e SECRET_KEY="your-secret" \
  -e CORS_ORIGINS="https://your-app.vercel.app" \
  -e DEMO_PASSWORD="your-password" \
  house-price-api
```

On Render: **New** â†’ **Web Service** â†’ connect repo â†’ **Docker** â†’ root `Dockerfile`.

Health check path: `/health`.

### Local full-stack (Docker API only)

```bash
docker compose up --build
```

Then run the frontend locally with:

```bash
cd frontend
echo VITE_API_BASE_URL=http://localhost:8000 > .env.local
npm run dev
```

## Step 2 â€” Deploy the frontend (Vercel)

1. [Vercel](https://vercel.com) â†’ **New Project** â†’ import the same repo.
2. **Root Directory:** `frontend`
3. **Environment variable:**
   - **`VITE_API_BASE_URL`** = your API URL from Step 1, **no trailing slash**  
     Example: `https://house-price-api.onrender.com`
4. Deploy.

---

## Step 3 â€” CORS

The browser only allows your Vercel origin if the API allows it.

On the API host, set **`CORS_ORIGINS`** to include your exact Vercel URL:

```text
https://your-project.vercel.app
```

Multiple origins: comma-separated, no spaces (or match how `backend/app/main.py` splits).

Redeploy or restart the API after changing `CORS_ORIGINS`.

---

## Step 4 â€” Sign in

Use the credentials from the API environment (`DEMO_EMAIL` / `DEMO_PASSWORD`), or the defaults in `backend/app/config.py` if you did not override them.

## Production notes

| Topic | Detail |
|-------|--------|
| **Ephemeral disk** | Uploads and trained `joblib` files may **disappear** on restart on free tiers. For real use, add object storage or a paid disk. |
| **Cold starts** | Free Render web services **spin down**; first request can take ~30â€“60s. |
| **Secrets** | Never commit `.env`; set variables in the host dashboard. |
| **HTTPS** | Use `https://` for both UI and API in `VITE_API_BASE_URL` and `CORS_ORIGINS`. |

---

## Troubleshooting

- **CORS errors** â€” `CORS_ORIGINS` must match the browser origin exactly (scheme + host, no path).
- **401 / network** â€” Check `VITE_API_BASE_URL` in Vercel rebuild after changing env vars.
- **Empty dashboard** â€” API must load `sample_data/houses.csv` on startup (full repo clone / Docker image includes `sample_data`).
